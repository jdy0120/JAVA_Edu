HTML(Hyper Text Markup Language)
 - 일종의 언어로 인터프리터 언어에 속하며, 웹 브라우저를 통해 실행됨
 - 웹 브라우저를 통해 html을 실행시키면 html의 문법대로 웹 문서를 보여줌
 - html은 태그라는 명령어들의 집합으로 <html>부터 </html>까지를 하나의 문서로 취급함
 - 단, html은 오류가 발생하지 않아 잘못 했을 경우 잘못을 찾는 것이 어렵다.
	오류가 나지 않고, 이상한 동작을 하게 되어 조심히 사용해야 함
 - 1990년 대 중반경 HTML 4.0이 나오고, 2010년 경 HTML 5.0이 나오게 됨


웹 표준화(Web Standard)
 - 2000년대 초반에 발표된 XHTML 1.0을 기반으로 웹에서 사용되는 언어들(HTML, CSS, JavaScript 등)을 표준화하여 지키자는 운동이 발생
	XHTML의 X는 XML의 X로 문법을 지켜서 HTML을 사용하자는 의미
 - 2000년대 후반에 본격적으로 진행되게 되는 웹 표준화로 인해 문법을 지킨 HTML등을 사용하게 됨
 - 웹 표준화는 모든 사이트가 모든 브라우저에서 동일하게 보이고, 동일하게 동작되게 하자는 의미
	모든 브라우저에서 동일하게 사용하는 것 외에 모든 사용자(시각장애인 포함)가 사이트를 원활하게 사용할 수 있게 하자의 의미도 가짐
	시각장애인 전용으로 리더기가 있어 사이트의 내용을 읽어줄 수 있음(단, html태그를 정확하게 사용했을 경우에만 리더기가 정확하게 읽어줄 수 있음)


태그(tag)
 - '<'로 시작하여 '>'로 끝나는 html의 명령어로 보통 여는 태그와 닫는 태그로 구분됨
	예) <여는 태그> ... </닫는 태그>
 - 대부분의 태그는 속성을 가지고 있어 태그에 여러가지 기능을 부여할 수 있음
 - 문법
	<태그명 속성명1="값1" 속성명2="값2" ... 속성명n="값n">
	값은 큰 따옴표로 묶어서 다른 속성이나 값들과 헷갈리지 않게 함
	단, 작은 따옴표나 아예 따오표를 생략해도 동작되나 될 수 있으면 큰 따옴표를 붙여 사용하는 것이 좋다.
	웹 표준화 작업 후 닫는 태그가 없는 단독 태그들도 자체적으로 닫는 의미를 사용함. 예) <태그명> -> <태그명 />
 - 종류
	인라인 요소(inline element) : 태그를 사용해도 줄이 바뀌지 않는 태그들의 총칭
	 - 인라인 요소안에서 블록 요소는 사용할 수 없다.
	블록 요소(block element) : 태그를 사용하면 다음 내용은 다음 줄로 줄이 바뀌게 하는 태그들의 총칭
	 - 독립적인 형태의 상자(box)를 의미하며 블록요소와 다른 블록요소 사이에는 약간의 여백이 존재하며 그 여백은 브라우저마다 조금씩 차이가 있을 수 있음(CSS에서 여백을 동일하게 조절한 후 사용해야 함)
	 - 블록 요소안에서 인라인 요소를 사용할 수 있다.
	 - 블록 요소의 옆에는 다른 요소들이 올 수 없으나 CSS를 이용하면 옆에 다른 요소가 올 수 있다.


공용 태그
1. <html> 태그
 - html문서의 시작과 끝을 의미하는 태그
 - <html>부터 </html>까지가 하나의 웹문서이나 범위 바깥에서도 입력하면 동작됨
 - 모든 html태그들은 기본적으로 <html>태그 내부에 위치해야 한다.
 - <html>태그 내부에는 <head>와 <body> 두 개의 태그가 존재함

2. <head> 태그
 - 해당 문서의 윗부분에 존재하면 화면에 보이는 것 보다 보이지 않는 작업이 더 많음
 - 검색을 용이하게 하기 위한 <meta>태그나 제목표시줄에 내용을 등록하는 <title> 등이 존재한다.
 - 필수는 아니지만 자바스크립트를 위한 <script>태그나 스타일시트를 위한 <style>태그를 사용하기도 함
	<script>와 <style> 모두 <body>태그 안에서도 사용할 수 있으나 보통은 보기 편하게 하기 위해 <head>태그 내에서 입력함
	경우에 따라 양이 너무 많으면 <script>태그는 </html>태그의 아래에 입력하기도 함

2-1. <title> 태그
 - <head>태그 내에 존재하는 태그로 브라우저 상단의 탭에 있는 페이지 제목을 입력받는 태그
 - 예전에는 쓰임새가 많았으나 탭 브라우저의 등장으로 보여줄 자리가 좁아 그 쓰임새가 축소된 태그

2-2. <meta> 태그
 - <head>태그 내에 존재하는 태그로 해당 웹문서의 설명 등에 정보를 저장하는 태그로 주로 검색용으로 사용됨
 - 여러 가지 속성을 이용하여 여러 종류의 값들을 저장하는 태그이나 검색에서 우리나라 포털에서는 큰 쓰임새는 없음

3. <body> 태그
 - 실제로 사용자에게 보여질 부분에 대한 영역으로 거의 대부분의 작업이 이루어지는 태그
 - <body> 태그는 기본적으로 여백을 가지고 있어 컨텐츠가 브라우저에 딱 붙지 않게 해줌
	단, 대부분 스타일시트를 사용하여 여백을 없앤후 사용하게 됨

4. <div> 태그
 - 블록 그룹화 태그로 분할, 나누기의 의미를 가짐
 - 주로 블록 요소를 그룹화하기 위해 사용되며 id와 class 속성으로 구분지어 사용할 수 있음
 - 레이아웃을 구현할 때 많이 사용됨

5. <span> 태그
 - 인라인 그룹화 태그로 주로 인라인 요소를 그룹화 할 때 사용함
 - div와 마찬가지로 id와 class 속성으로 구분지어 사용할 수 있음
 - 인라인 요소이므로 블록요소(div, p, ul 등)를 포함할 수는 없음


공용 속성
 - 대부분의 태그에서 사용할 수 있는 속성으로 보통 범위를 갖는 태그에서 사용함
	범위를 갖는 태그 : 열고 닫는 모든 태그와 img태그, input태그를 의미

 - title 속성 : 마우스를 올렸을 때 툴팁처럼 문자열을 잠깐 보여주는 기능

 - style 속성 : 해당 태그의 범위에 스타일을 직접 지정하는 속성

 - name 속성 : 지정한 범위(태그)에 특정 이름을 부여하여 CSS나 JS에서 구분자로 사용
	주로 컨트롤의 이름으로 많이 사용되며, 동일한 이름이 있으면 배열로 인식하여 동작됨

 - class 속성 : 지정한 범위에 특정 class명을 부여하여 CSS나 JS에서 구분자로 사용
	class속성의 값은 한 페이지내에서 동일한 값이 여러 개 존재해도 상관없음
	같은 class명을 가진 범위들을 하나로 묶어주는 역할을 함

 - id 속성 : 지정한 범위에 특정 id명을 부여하여 CSS나 JS에서 구분자로 사용
	id속성의 값은 한 페이지내에서 유일한 값이어야 함
	유일한 값이어야 하지만 동일한 값이 존재해도 오류가 발생하진 않음
