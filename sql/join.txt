조인(join)
 - 두 개 이상의 테이블을 서로 묶어서 하나의 결과 집합으로 만들어 내는 기능
 - 조인을 이용하면 속도가 느려지지만 조인을 이용하지 않고 처리하기 위해서는 더 느려지므로 조인기능을 사용해서 작업해야 하는 곳이 반드시 있음

1. Inner Join
 - 내부 조인이라고도 하며 가장 많이 사용되는 조인으로 일반적으로 조인이라 하면 Inner Join을 의미하는 경우가 대부분임
 - 문법
select 가져올 필드(들) from 테이블1 inner join 테이블2 on 테이블1과 2의 결합 조건
또는
select 가져올 필드(들) from 테이블1, 테이블2 where 테이블1과 2의 결합 조건

 - 가져올 필드의 이름이 두 테이블 모두에 있을 수도 있으므로 테이블명.컬럼명의 방식으로 작업해야 하나 테이블명이 길 경우 번거로우므로 테이블에 alias를 붙여 사용하는 것이 일반적임
 - 조인의 조건은 보통 관계를 맺지 않아도 가능하나 일반적으로 관계를 맺은 PK와 FK를 비교하는 것으로 조건을 충족시킴
 - 세 개이상의 테이블 조인
select 가져올 필드(들) from 테이블1 inner join 테이블2 on 테이블1과 2의 결합 조건
inner join 테이블3 on 테이블3와 테이블1 또는 테이블2의 조건

select 가져올 필드(들) from 테이블1, 테이블2, 테이블3 where 테이블1과 2의 결합 조건 and 테이블3과 테이블1 또는 테이블2의 조건

2. Outer Join
 - 조인의 조건에 만족하지 않는 레코드들도 포함시키는 조인방법
 - 문법
select 가져올 필드(들) from 테이블1 <LEFT | RIGHT> [outer] join 테이블2 on 테이블1과 테이블2의 결합 조건
2의 결합 조건
 - left를 사용하면 왼쪽의 테이블인 테이블1의 레코드는 모두 출력하고, 테이블2의 레코드는 조건에 맞는 레코드들만 출력
 - right를 사용하면 오른쪽의 테이블인 테이블2의 레코드는 모두 출력하고, 테이블1의 레코드는 조건에 맞는 레코드들만 출력
 - outer는 생략해도 됨
