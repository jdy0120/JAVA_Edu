학원에서는 수강하는 과정별로 계속 새롭게 원생 데이터를 등록해야 함
1. 과목 테이블이 따로 존재하여 과목에 대한 정보를 저장해 놓는지?
2. 과정 테이블에서 과목에 대한 내용을 직접 입력할지?

같은 과목이라도 수업시간이나 내용에서 차이가 날 수 있음
같은 과목도 내용 등에서 차이가 날경우 다른 과목으로 취급하여 각각 따로 등록해 놔야 함

과정관리에서 과정 등록시 과목과 강의실 등의 시설/장비 그리고, 강사에 대한 선택은 따로 먼저 등록되어 있어야 함

과정테이블과 원생테이블의 관계가 1 대 다의 관계이므로 원생테이블에 저장할 수 있음

과정 등록시 수강생 입력 때문에 느려지는 걸 막을 수 있음
단순히 수강생 ID만을 보여줄 경우 조인을 할 필요가 없어 속도가 빠름

과정 테이블에 들어있는 값들이 모두 ID이므로 다른 원생 정보를 보려면 다시 원생 테이블에 연결하여 쿼리를 실행해야 함

String[] arrID = 수강생.split(", ");
for (int i = 0 ; i < arrID.length ; i++) {
	query = "select * from 원생테이블 where 원생ID = '" + arrID[i] + "'";
}
과정 테이블 안에 있는 수강생 데이터는 PK이므로 select 쿼리로 데이터를 가져올 때 하나씩만 가져올 수 있음
수강생 데이터에 들어있는 원생ID의 수 만큼 쿼리를 작업해야 함

과정 내에서 수강생 목록을 보지 않을 경우에는 큰문제는 없음
단, 목록은 거의 대부분의 과정에서 보여주므로 신경써야 함

1 대 1이나 1 대 다의 관계에서는 그대로 작업해도 되지만 다 대 다의 관계에서는 정규화를 해야 함
